| 2026-06-01 | **Fix failing chunks: allow `external_directory` reads (LADR-025).** Chunks failed with `## ⚠️ Review Failed for Chunk` because the prompt tells the model to `read_file` in-repo dot-path context files (`.github/*AGENTS.md`, `.docs/nfr/*`, `.ai/rules-scoped/*`), but opencode's `external_directory` permission defaults to `ask` → headless `opencode run` auto-rejects → the read errors → empty chunk → fail-closed forces REQUEST_CHANGES on clean PRs. Regression from dropping gemini-cli's `--yolo` (LADR-023). | 2026-05-29 | **Fix local "deadlock" on an unreachable gateway.** When the LiteLLM gateway (`*.private.prod`) was unreachable (e.g. not on VPN), every `opencode run` hung indefinitely and the review never progressed. Two fixes in `local-review.sh`: (1) a **gateway reachability preflight** — curls `$OPENCODE_LITELLM_URL/health` with an 8s timeout before any review work and aborts with "connect to VPN" guidance instead of hanging; (2) **rewrote the macOS `timeout` shim** to run the command in its own process group and `kill -KILL` the whole group on expiry. The previous `perl -e 'alarm; exec bash …'` shim never worked — bash traps SIGALRM and the `opencode` grandchild was orphaned and kept running, so the 60s/300s timeouts bounded nothing and leaked processes. | BNKI-001 |
| 2026-05-29 | **Zero-setup local runs.** `local-review.sh` now bootstraps its own credentials: when `OPENCODE_LITELLM_URL`/`OPENCODE_LITELLM_API_KEY` aren't already in the environment it harvests their allowlisted `export` lines from `~/.zshenv|.zprofile|.zshrc|.bash_profile|.bashrc|.profile` — non-`eval` value parsing so URL/query chars don't break it. This fixes the `/ai-review:local-review` skill, whose Haiku subagent runs a non-interactive shell that never sources `.zshrc`. The skill's Step 0 no longer pre-gates on credentials (tools only) — the script resolves + validates them. `setup-opencode-config.sh` now self-heals: an installed `litellm-gemini` config still referencing the old `{env:GEMINI_API_KEY}`/`{env:GOOGLE_GEMINI_BASE_URL}` names is refreshed to the committed `OPENCODE_LITELLM_*` version (hand-rolled personal configs without a litellm-gemini block are still left untouched). | BNKI-001 |

| 2026-05-28 | **Migration: `@google/gemini-cli` → `opencode` CLI (LADR-023).** Public-tier Gemini CLI reaches EOL on 2026-06-18; we use the public-tier endpoint behind an internal LiteLLM proxy. Trade-off: LADR-015 file-state verification is currently unavailable since no agent is passed — chunk reviews verify against diff context only; restore via a custom narrow-tool agent in `opencode.json` is a tracked follow-up. Empty/tiny chunk output now produces a visible `## ⚠️ Review Failed for Chunk:` marker in the posted review body. CI workflow: `GEMINI_API_KEY` + `GOOGLE_GEMINI_BASE_URL` promoted to job-level env; opencode install + config + DB warmup (`opencode stats`) consolidated into one post-checkout step.
